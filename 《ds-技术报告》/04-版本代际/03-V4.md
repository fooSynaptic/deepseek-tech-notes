## 核心结论摘要

- V4 preview 面向 **1M context** 与 Agentic Coding；Pro 1.6T/49B，Flash 284B/13B。
- **CSA 4:1 + HCA 128:1** 混合压缩注意力；**mHC** 双随机流形超连接。
- **Hash MoE + FP4**、**Muon** 优化器同步翻新，不利于单变量 ablation。
- KV 异构（CSA/HCA/SWA/Indexer/Tail），配套 **DSpark / HiSparse / 磁盘 Prefix Cache**。

---

## 定位

2026 年 preview release，面向 **百万 token 上下文**与 **Agentic Coding**（100K–1M token 代码库、多轮 tool trace）。相对 V3.2 是**架构级大步进**：注意力、残差、优化器、MoE 路由、量化同时翻新，不利于做单一变量 ablation。

两个规格：

| 模型 | 总参数 | 激活参数 | 定位 |
|------|--------|----------|------|
| **V4-Pro** | 1.6T | 49B | 能力上限，含 Pro-Max 推理模式 |
| **V4-Flash** | 284B | 13B | 效率优先 |

| V4-Flash @ 1M | 10% | 7% |

## 推理 infra 关注点

V4 的 cache **异构**，不再是单一 MLA latent：

| KV 类型 | 特点 |
|---------|------|
| CSA 压缩 entry | 4:1，稀疏 top-$k$ |
| HCA 压缩 entry | 128:1，dense |
| SWA | 滑动窗口，独立 eviction |
| Indexer KV | CSA lightning indexer |
| Tail buffer | 未凑满压缩块的尾 token |

**KV layout**→ **[专文：V4 KV Layout](../06-推理基础设施/05-V4-KV-Layout.md)**（Classical + State 双池；演进总览 [§5.3](../01-总览/01-版本演进总览.md#v4-kv-layout)）

**HiSparse** → **[专文：V4 HiSparse](../06-推理基础设施/06-V4-HiSparse.md)**（inactive C4 offload；[§5.3](../01-总览/01-版本演进总览.md#v4-hisparse)）

**磁盘 Prefix Cache**→ **[专文：V4 磁盘 Prefix Cache](../06-推理基础设施/07-V4-磁盘Prefix-Cache.md)**（SWA 三档策略；[§5.3](../01-总览/01-版本演进总览.md#v4-disk-prefix-cache)）

**Decode 吞吐**：V4 预览引擎 → **[投机解码与 DSpark 专文](../06-推理基础设施/04-DSpark投机解码.md)**（含 MTP、MTP-1 基线、DSpark；与 HiSparse **正交**）。

> V4 的 KV-offload **[与 V3.2 ESS 完全不同](../01-总览/06-基础设施线导读.md#3-节点间关系一句话)**，需围绕异构压缩 cache 重新设计内存层级。
